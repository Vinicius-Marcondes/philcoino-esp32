# PRD-008 — Live temperature graph clarity and CSV relocation

Status: Active

Approved: 2026-07-20

## Summary

Simplify the Dashboard temperature graph to its horizontally paged thirty-second
Live view, move current-day CSV export to Machine, and use an adaptive labeled
Y-axis so nearby brew temperatures are visually distinguishable.

This follow-up supersedes PRD-007's requirement to preserve the Today graph. It
does not change history capture, synchronization, firmware authority, or safety
boundaries.

## Goals

- Keep the graph focused on precise live and recent-window inspection.
- Make values such as 95°C visibly distinct from the 100°C grid line.
- Place observational data export with other device-management information.
- Preserve complete current-day storage, backfill, and CSV fidelity.

## Non-Goals

- Changing the thirty-second page duration, retention, CSV columns, SQLite
  schema, protocol, simulator, firmware, or control behavior.
- Removing access to earlier Live pages or changing gap semantics.
- Treating historical readings or commands as physical-state evidence.

## Context

The Today overview compresses too many readings for the available mobile graph
area. The current warm graph also uses a broad 70–100°C range with only its
endpoints labeled, which makes a 95°C trace appear to be at 100°C.

## Requirements

- Remove the Live/Today selector and Today-only downsampling while retaining
  thirty-second paging, earlier-page inspection, follow-latest, jump-to-latest,
  and explicit gaps.
- Calculate each visible page's Y-axis from boiler and active-target values,
  with 2°C padding, at least a 10°C range, and five rounded evenly spaced ticks.
- Use the same scale and plot coordinates for labels, grid lines, boiler data,
  and target data.
- Continue storing and backfilling the complete current local day and export all
  stored rows with the existing CSV columns and ordering.
- Put a localized export card after Machine's uptime/Steam Timer metrics and
  before Saved Machine. Keep it usable with stored data while offline.
- Keep graph storage/synchronization feedback separate from export-operation
  feedback and preserve accessible disabled/loading/error states.

## Implementation Overview

1. Replace Live/Today graph state with a Live-only pager and shared adaptive
   scale helpers.
2. Move export presentation and its error state to a dedicated Machine card.
3. Remove obsolete Today presentation code, align localization and current
   architecture documentation, and verify mobile behavior.

## Acceptance Criteria

- [x] Dashboard exposes only the thirty-second Live graph and retains paging,
  gap, and follow-latest behavior.
- [x] A 95°C value aligns with a labeled adaptive grid position and remains
  visually distinct from 100°C.
- [x] Machine orders Steam Timer, CSV export, and Saved Machine as specified,
  and export remains available from stored history while offline.
- [x] CSV still includes every stored current-day row with unchanged columns.
- [x] All mobile tests, typecheck, and lint pass.
- [ ] Human accepts graph readability, page swiping, card placement, and
  English/Portuguese layout on an iPhone.

## Open Questions

- No product decision remains open. Native visual acceptance remains a Human
