# PRD-009 — Landscape mounted-control experience

Status: Active

Approved: 2026-07-20

## Summary

Support portrait and both landscape directions across the native mobile app so
a phone can serve as a mounted espresso-machine control screen. Landscape uses
a compact three-dot gesture rail and responsive content while preserving every firmware,
protocol, polling, and acknowledged-mutation boundary.

## Goals

- Keep live state, temperature, the thirty-second graph, and quick extraction
  controls visible together on a typical landscape phone.
- Make Pairing, Dashboard, Profiles, and Machine usable in either orientation.
- Let the user persist an opt-in preference that keeps paired screens awake.

## Non-Goals

- Kiosk mode, forced orientation, tablet-specific design, or changing OS
  rotation-lock behavior.
- Protocol, simulator, firmware, history, authentication, or control changes.
- Treating the mounted app as part of the heater or pump safety loop.

## Context

The app is currently locked to portrait. Its paired presentation is a single
vertical column with bottom navigation, so rotating the phone would not make
effective use of the wider mounted-screen shape.

## Requirements

- Allow automatic portrait and landscape rotation and respect safe areas in
  either landscape direction.
- Keep portrait behavior intact. In landscape, replace bottom navigation with
  a narrow transparent three-dot rail; tapping a dot or swiping vertically
  anywhere changes pages. Taller pages retain normal scrolling until a swipe
  reaches the matching top or bottom boundary. Page changes use a short,
  direction-aware fade-and-slide transition. Keep the rail compact beside the
  plain edge and apply its leading safe-area inset when the notch is beside it.
  Use compact, multi-column presentation. The live Dashboard places
  equal-height status, cooldown, and extraction cards in its top row, then
  balances temperature and the graph at a one-third/two-thirds ratio beneath.
  The compact quick-profile chooser opens as an overlay without resizing that
  row, with Manual spanning its first row and the four profile slots arranged
  2-by-2 beneath it. Compact landscape omits the pump-command copy, places a
  stable contextual Profiles or Machine action beneath the selector, and lets
  Start or Stop fill the adjacent column without adding multiline card copy.
  Portrait keeps the full pump status and blocker detail. Profiles gives the
  local editor the left half, then stacks the 2-by-2 profile chooser above sync
  in the right half. Compact duration steppers use unified rounded controls.
- Preserve graph paging, page scroll restoration, active-extraction access,
  keyboard handling, fault prominence, accessible touch targets, and localized
  English and Portuguese copy.
- Default Keep screen awake to off. Persist it independently of the selected
  machine and apply it on any paired foreground screen in either orientation.
- Fail preference loading or saving to off and show non-blocking feedback.
- Never restart polling, duplicate mutations, or present requested state as
  acknowledged state because of rotation.

## Implementation Overview

1. Add pure responsive layout classification and enable automatic orientation.
2. Adapt Pairing and paired pages with landscape columns and side navigation.
3. Add strict local display preferences and an Expo keep-awake lifecycle guard.
4. Align localization, tests, and mobile architecture documentation.

## Acceptance Criteria

- [ ] Portrait and both landscape directions work on iOS and Android phones.
- [ ] A typical landscape phone initially shows machine state, temperature,
  graph, and quick extraction controls without clipping at standard text size.
- [ ] Pairing, Profiles, and Machine use responsive landscape columns and remain
  scrollable with the keyboard or larger accessibility text.
- [ ] The dot rail preserves all three destinations and indicates an active
  extraction on Dashboard; graph swipes do not switch destinations.
- [ ] Keep screen awake defaults off, persists across restart and machine
  removal, and releases when disabled, unpaired, backgrounded, or unmounted.
- [x] Mobile tests, typecheck, and lint pass.
- [ ] Human accepts native layout, rotation, graph gestures, and English and
  Portuguese copy on representative iOS and Android phones.
