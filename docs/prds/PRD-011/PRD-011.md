# PRD-011 — Brew by weight

Status: Active

Approved: 2026-07-23

## Summary

Add a firmware-authoritative scale using a 1 kg load cell and HX711. A custom
profile can start in its existing timed form or with an acknowledged weight
target. Manual remains unchanged. Weight Start automatically tares a stable
scale before the pump is commanded and requires no second confirmation.

## Goals

- Stop a profile at a compensated beverage target with 0.1 g resolution.
- Keep calibration, tare, measurement validation, fallback, and pump timing in
  firmware after Start acknowledgement.
- Add guided calibration, live diagnostics, per-profile defaults, 90-day local
  shot history, and CSV export to mobile.
- Preserve existing API v1 and API v2 Manual/timed response shapes.

## Non-Goals

- Weight control for Manual extraction.
- Certified scale accuracy, flow or pressure measurement, adaptive drip
  prediction, cloud history, or firmware-persisted shot history.
- Detecting whether a cup is physically present before automatic tare.
- Treating software, simulator, or low-voltage evidence as energized approval.

## Context

The existing firmware owns profile timing and the GPIO10 pump command. The
planned HX711 uses 3.3 V, channel A, GPIO0 for data, and GPIO1 for clock. The
expected platform, cup, and beverage total is about 144 g, below the cell's
1 kg rating. Displaying tenths of a gram does not establish matching accuracy.

## Requirements

- Accept profile weight targets from 5.0–100.0 g and compensation from
  0.0–10.0 g, with compensation strictly below the target.
- Persist a two-point calibration using an unloaded platform and a stable
  50.0–500.0 g reference. Keep automatic per-shot tare volatile.
- A weighted Start waits up to three seconds for stability, tares, then starts.
  Failure leaves the pump off and returns a retryable scale error.
- Complete on `target - compensation` in any profile phase. Continue sampling
  for at most ten seconds after pump-off to capture settled final weight.
- If scale feedback fails, use the selected profile's original monotonic
  deadline, warn the app, and reject another weighted Start until acknowledged.
- Keep the independent 60-second cutoff. Scale state never changes heater fault
  policy and never affects Manual or ordinary timed extraction.
- Add authenticated scale read, calibration, cancel, and warning-acknowledgement
  operations without changing existing state/profile/extraction responses.
- Add a Scale mobile page plus Dashboard timed/weight selection. Store defaults
  and 90-day summaries locally and support clear plus UTF-8 CSV export.

## Implementation Overview

1. Extend OpenAPI/Zod with additive scale and weighted-Start types.
2. Add deterministic simulator measurement, calibration, cutoff, and fallback.
3. Add host-testable HX711/calibration policies and ESP-IDF adapters, then bind
   them to extraction control without blocking heater/pump timing.
4. Add mobile diagnostics, guided calibration, weighted Start, history, export,
   localization, and responsive navigation.
5. Align safety/hardware documentation and record automated and low-voltage
   evidence separately.

## Acceptance Criteria

- [ ] Existing Manual/timed requests and strict responses remain compatible.
- [ ] Pump cannot start before successful automatic tare.
- [ ] Weighted cutoff, fallback deadline, acknowledgement gate, idempotency, and
  the independent cutoff are deterministic in simulator and firmware tests.
- [ ] Calibration survives reboot while tare and active/terminal scale state do
  not resume as an active workflow.
- [ ] Mobile exposes the complete responsive workflow, 90-day history, clear,
  and CSV export without presenting requested state as acknowledged.
- [ ] Protocol, simulator, mobile, firmware host, and target-build checks pass.
- [ ] Disconnected low-voltage evidence covers pins, cadence, calibration,
  automatic tare, stability, and injected disconnection.

## Open Questions

- No product decision remains open. Physical wiring and energized compensation
  tuning remain separately supervised acceptance work.
